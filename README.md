# Synthea EHR Data Pipeline

A production-style data pipeline that ingests synthetic FHIR R4 patient records, applies data quality checks, loads them into a SQL warehouse, and surfaces clinical insights.

## What It Demonstrates

| Skill | How |
|---|---|
| **FHIR / HL7 formats** | Parses FHIR R4 Bundle NDJSON; extracts Patient, Condition, Encounter, Observation resources |
| **Clinical data modeling** | SNOMED CT (conditions), LOINC (labs), ICD-adjacent encounter classes |
| **ETL pipeline design** | Modular generate → parse → quality-check → load → analyze stages |
| **Data quality frameworks** | Completeness, referential integrity, value-range, duplicate-key checks with PASS/WARN/FAIL grading |
| **SQL warehousing** | Normalized schema with indexes + analytical views (patient summary, diabetes cohort, A1c trend) |
| **Python best practices** | Typed functions, pytest unit tests, CLI args, parquet intermediate storage |

## Quickstart

```bash
pip install -r requirements.txt
python run_pipeline.py               # generates 200 patients, runs all stages
python run_pipeline.py --patients 1000
```

Output:
- `data/ehr_warehouse.db` — queryable SQLite warehouse
- `data/quality_reports/quality_report.json` — DQ report
- `data/processed/*.parquet` — flat tables (patients, conditions, encounters, observations)

## Pipeline Stages

```
generate_data.py   →  synthea_fhir_bundles.ndjson   (FHIR R4 NDJSON)
parse_fhir.py      →  *.parquet                      (flat DataFrames)
data_quality.py    →  quality_report.json            (DQ checks)
load_to_sqlite.py  →  ehr_warehouse.db               (SQL warehouse)
analytics.py       →  printed findings               (clinical insights)
```

## Key Findings (200 patients)

Running the pipeline surfaces findings like:
- Condition prevalence by SNOMED code
- Emergency utilization rate by year
- Average A1c: diabetic vs. non-diabetic cohort
- High-utilizer patient identification (top 10% by encounter count)
- Lab completeness by encounter type

## Tests

```bash
python -m pytest tests/ -v
```

## Data Source

Uses a built-in FHIR R4 synthetic generator (no external dependencies). Replace `generate_data.py` with the [Synthea CLI](https://github.com/synthetichealth/synthea) for more realistic distributions:

```bash
# Optional: use real Synthea output
java -jar synthea-with-dependencies.jar -p 1000 --exporter.fhir.export true
# Then place the exported FHIR bundles in data/raw/ and run:
python run_pipeline.py --skip-generate
```
